实时数据**不入 Chroma**。

## 工具设计

### 1. `weather_query`

| 项 | 说明 |
|----|------|
| 数据源 | [Open-Meteo](https://open-meteo.com/) |
| API Key | 不需要 |
| 参数 | `location: str` |
| 意图词 | 天气、气温、温度、湿度、风力、预报 … |

### 2. `news_search`

| 项 | 说明 |
|----|------|
| 数据源 | `.env` `NEWS_RSS_FEEDS`（默认 BBC + 中新网） |
| 参数 | `query?: str` |
| 意图词 | 新闻、头条、热点、资讯、快讯、报道 |

### 3. `sports_query`（新增）

| 项 | 说明 |
|----|------|
| 数据源 | [TheSportsDB](https://www.thesportsdb.com/) 免费 API |
| API Key | 默认测试 Key `3`（`SPORTSDB_API_KEY`） |
| 参数 | `sport: Soccer/Basketball`，`team?: str` |
| 逻辑 | 有球队名 → `searchevents`；否则 → `livescore` |
| 意图词 | 体育、足球、篮球、比分、赛况、比赛、联赛、NBA、英超 … |

**说明：** `sports` 意图排在 `news` 之前，「体育新闻」优先走体育 Tool。

## SSE 流式 API

### 端点

`POST /agent/stream` — 请求体与 `/agent` 相同：

```json
{ "message": "北京天气怎么样", "session_id": "optional-uuid" }
```

### 响应

`Content-Type: text/event-stream`，每行：

```
data: {"type":"token","content":"北"}

```

### 事件类型

| type | 时机 | 字段 |
|------|------|------|
| `workflow` | 意图判定后 | `workflow` |
| `plan` | 工具计划 | `plan` |
| `tool_calls` | 工具执行完 | `tool_calls` |
| `token` | LLM 每个片段 | `content` |
| `done` | 结束 | `answer`, `sources`, `session_id` … |
| `error` | 异常 | `detail` |

**注意：** 工具调用在流式开始前**同步完成**（通常 <2s），仅 LLM 总结阶段流式输出。

### 代码落点

| 文件 | 职责 |
|------|------|
| `app/core/llm.py` | `chat_messages_stream()` |
| `app/agent/executor.py` | `run_agent_stream()`, `format_sse_event()` |
| `app/api/agent.py` | `POST /agent/stream` |
| `frontend/src/api.js` | `sendAgentMessageStream()` |
| `frontend/src/App.jsx` | token 累加 + 光标动画 |

## 测试命令

```powershell
# 健康检查（0.3.2）
Invoke-RestMethod http://127.0.0.1:8000/health

# 同步（兼容）
Invoke-RestMethod -Uri http://127.0.0.1:8000/agent -Method Post `
  -ContentType "application/json" -Body '{"message":"足球比赛比分"}'

# 流式（PowerShell 需逐行读 SSE）
curl -N -X POST http://127.0.0.1:8000/agent/stream `
  -H "Content-Type: application/json" `
```
