# Day21_1 — Web 聊天界面设计（Frontend）

> 版本：**v0.3.0+** | 技术栈：**React 18 + Vite 5**  
> **定位：简单可用的企业 Agent Web UI，对接现有 FastAPI 后端**

## 学习目标

- [ ] 理解前端与 `POST /agent`、`POST /upload` 的对接方式
- [ ] 掌握页面布局：顶栏 + 侧栏上传 + 主聊天区
- [ ] 能在助手回复中展示 **引用来源** 与 **工具 / Workflow 状态**
- [ ] 能本地启动前后端并联调

---

## 今日边界（重要）

| 做 | 不做 |
|----|------|
| 单页聊天 + PDF 上传 | 用户登录 / 权限 |
| 展示 workflow / plan / sources | 流式 SSE 打字机效果 |
| `session_id` 多轮记忆（localStorage） | 入库脚本一键化（仍用命令行） |

**Day21_1 核心成果：浏览器里能聊、能传 PDF、能看见 Agent 决策与引用。**

```
┌──────────────┐     HTTP      ┌─────────────────────┐
```

## 区域说明

### 1. Header（顶栏）

| 元素 | 说明 |
|------|------|
| 标题 | `AI Project Assistant` |
| 副标题 | 能力一句话：聊天 / 上传 / 来源 / 工具 |
| 后端状态 | `GET /health` → 绿标 `后端 0.3.0` 或红标 `后端未连接` |
| 新会话 | 重新生成 `session_id`，刷新页面，清空当前聊天记录 |

### 2. Sidebar（左侧）

| 元素 | 说明 |
|------|------|
| 上传按钮 | `<input type="file" accept=".pdf">`，调用 `POST /upload` |
| 上传中 | 按钮文案变为「上传中…」，禁用重复点击 |
| 成功反馈 | 聊天区插入 **系统消息**：`已上传 PDF：xxx（8MB）。请执行入库脚本…` |
| 会话 ID | 显示前 8 位，便于调试 Memory |

### 3. Chat Area（主区）

| 元素 | 说明 |
|------|------|
| 空状态 | 示例问题列表（总结 PDF / README / 计算） |
| 消息列表 | 用户右对齐、助手/系统左对齐，自动滚到底部 |
| 加载态 | 助手气泡「思考中…」（请求进行中） |
| 错误条 | 上传或发送失败时，输入框上方红色提示 |
| 输入区 | 多行文本框 + 发送按钮；`Enter` 发送，`Shift+Enter` 换行 |

### 4. MetaBlock（助手元数据，嵌在气泡内）

仅 **助手成功回复** 展示，数据来自 `POST /agent` 响应：

| 区块 | 字段 | 展示形式 |
|------|------|----------|
| 工作流 | `workflow.intent` | 绿色 chip，如 `rag`、`filesystem`、`chat` |
| 工作流 | `workflow.need_tool` | chip：`已调用工具` / 直答时由 intent 体现 |
| 工作流 | `workflow.route` / `reason` | 灰色说明文字 |
| 工具计划 | `plan[]` | 每步：紫色 chip `tool` + `reason` + `args` JSON |
| 工具调用 | `tool_calls[]` | 紫色 chip 列表 |
| 引用来源 | `sources[]` | 列表：`{source} · Page {page} · chunk {chunk}（score）` |

展示约定与后端一致：`{source}  Page {page}`（见 `api.md`）。

## 与后端 API 对接

### 聊天（主链路）

```
用户输入 → POST /agent
  Body: { "message": "...", "session_id": "<uuid>" }
  Response:
    answer          → 气泡正文
    workflow        → MetaBlock 工作流区
    plan            → MetaBlock 工具计划
    tool_calls      → MetaBlock 工具调用
    sources         → MetaBlock 引用来源
    session_id      → 后端 Memory（前端已本地持久化）
```

`session_id`：首次访问 `crypto.randomUUID()`，存入 `localStorage.agent_session_id`。

### 上传

```
选择 PDF → POST /upload (multipart, field: file)
  Response: { filename, size }
  → 系统消息提示 + 提醒运行入库脚本
```

### 健康检查

```
页面加载 → GET /health
  Response: { status: "OK", version: "0.3.0" }
  → 顶栏状态徽章
```

## 交互流程

### 发送消息

```
1. 用户输入 → 点击发送 / Enter
2. 立即追加 user 气泡，清空输入框
3. loading=true → 显示「思考中…」
4. POST /agent
5. 成功 → assistant 气泡 + MetaBlock
6. 失败 → assistant 错误气泡 + error-bar
7. loading=false
```

### 上传 PDF

```
1. 选择文件 → uploading=true
2. POST /upload
3. 成功 → system 气泡 + 文件名/大小
4. 失败 → error-bar
5. uploading=false，清空 file input
```

### 新会话

```
1. 新 UUID 写入 localStorage
2. location.reload() → 消息列表清空，Memory 后端也用新 session
```

## 启动与构建

```powershell
# 终端 1：后端
python -m uvicorn app.main:app --reload --host 127.0.0.1 --port 8000

# 终端 2：前端
cd frontend
npm install
npm run dev
# → http://127.0.0.1:5173

# 生产构建
npm run build
# → frontend/dist/
```

环境变量（可选）：`VITE_API_BASE=http://127.0.0.1:8000`（不走代理时）。

## 常见坑

1. **只开前端未开后端** — 顶栏红标，发送报网络错误
2. **上传后无法 RAG** — 需手动跑入库脚本，UI 仅负责上传到 `uploads/`
3. **README 问题走 chat** — 检查 `.env` 中 `MCP_ENABLED=true` 且已重启 uvicorn
4. **CORS 错误** — 确认 `main.py` 已加 CORS，或只用 Vite 代理（同源）
5. **PowerShell 链式命令** — 用 `;` 分隔，不用 `&&`
